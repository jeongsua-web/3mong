  2. 인증 시스템 (Spring Security + JWT)

  "이 사람이 진짜 우리 앱 유저인지 확인하는 문지기"

  ① 사용자가 Google 로그인
  ② AWS Cognito가 JWT 토큰 발급  (= 입장권)
  ③ 모든 API 요청마다 토큰 검사
  ④ 토큰 안의 google_id로 "누구인지" 식별

  토큰 없이 API 호출하면 401 에러로 차단됨.
  Spring Security의 SecurityFilterChain에 JWT 전용 필터를 등록해서 구현
  모든 API 요청이 들어올 때마다 HTTP Header에 담긴 JWT 토큰을 추출하고, 이를 **AWS Cognito의 공개키(JWK)**와 대조하여 유효성을 검증하는 방식

  상세 로직 (Flow):

OncePerRequestFilter를 상속받은 커스텀 필터를 만듭니다.

Authorization 헤더에서 Bearer [Token] 값을 가져옵니다.

이 토큰이 변조되지 않았는지, 만료되지 않았는지 확인합니다.

검증이 완료되면 해당 유저의 정보(Email, ID 등)를 SecurityContext에 저장하여 컨트롤러에서 바로 쓸 수 있게 합니다.
