## 1. Project Overview (프로젝트 개요)

### 한 줄 정의
> **Fluento**는 AI 캐릭터와 **100% 영어 채팅**을 통해 사용자의 영어 실력을 자동으로 판단하고 **난이도를 실시간으로 조절**해주는 AI 기반 영어 학습 플랫폼이다.

### 핵심 마일스톤 (현재까지 진척도)

| 단계 | 항목 | 상태 |
|------|------|------|

| 설계 | API 명세서 작성 (JDK 25 기반) | ✅ 완료 |
| 설계 | DB 스키마 설계 | ✅ 완료 |

## 2. System Architecture & Tech Stack (시스템 아키텍처)

### 전체 시스템 구조

```
[Client (Mobile/Web)]
        │  HTTPS
        ▼
[AWS Cognito]  ←─── Google OAuth 2.0
   JWT 발급
        │
        ▼
[EC2 - Spring Boot 백엔드]
   ├── REST API (채팅방/유저/인증)
   ├── SSE 스트리밍 (AI 응답)
   ├── Redis (레벨 조절 스코어 캐시)
   ├── PostgreSQL (영구 데이터 저장)
   └── OpenAI GPT-4o API 연동
        │
        ▼
[AWS S3] ←── 이미지 메시지 저장 (Pre-signed URL)
```

### 기술 스택

| 영역 | 기술 | 선택 이유 |
|------|------|----------|
| Language | Java 17 + **Virtual Threads** | 높은 동시성 처리 (SSE 스트리밍 최적화) |
| Framework | Spring Boot 4.0 / Spring Framework 7 | 최신 LTS, WebFlux SSE 지원 |
| Build | Gradle 9 | 빠른 빌드, 최신 DSL |
| DB | PostgreSQL 18 + JPA + Flyway | 스키마 버전 관리, 안정적 RDBMS |
| Cache | Redis (Lettuce) | 레벨 조절 점수 이력 실시간 캐싱 |
| Auth | Spring Security + AWS Cognito | Google 소셜 로그인, JWT 검증 |
| AI | OpenAI GPT-4o | 대화·평가·레벨 판단 통합 처리 |
| Streaming | SSE (Server-Sent Events) | 단방향 실시간 스트리밍, 구현 단순 |
| 인프라 | AWS (EC2, S3, Cognito) | 확장성, 관리형 인증 서비스 |

> **JDK 25 선택 이유**: Virtual Threads (Project Loom) 활용으로 SSE 스트리밍 처리 시 스레드 블로킹 없이 높은 동시 접속 처리. Record Classes로 DTO 코드 간결화.

## 3. Core API Specifications (주요 API 명세)

**Base URL**: `https://api.fluento.com/api/v1` | 인증: `Authorization: Bearer {JWT}`

### 채팅방 관리

| 메서드 | 엔드포인트 | 기능 |
|--------|-----------|------|
| `POST` | `/chat/rooms` | 채팅방 생성 (캐릭터 1개당 1개 제한) |
| `GET` | `/chat/rooms` | 채팅방 목록 (고정/전체 필터, 페이지네이션) |
| `PUT` | `/chat/rooms/{roomId}` | 제목 수정 |
| `PUT` | `/chat/rooms/{roomId}/pin` | 고정/해제 토글 |
| `DELETE` | `/chat/rooms/{roomId}` | 채팅방 삭제 |

### 메시지 & AI 응답

| 메서드 | 엔드포인트 | 기능 |
|--------|-----------|------|
| `POST` | `/chat/rooms/{roomId}/messages` | 사용자 메시지 전송 (텍스트/이미지) |
| `GET` | `/chat/rooms/{roomId}/messages` | 메시지 히스토리 조회 |
| `GET` | `/chat/rooms/{roomId}/stream` | **SSE AI 응답 실시간 스트리밍** |
| `POST` | `/chat/rooms/{roomId}/initial-message` | AI 첫 인사말 요청 |
| `GET` | `/chat/rooms/{roomId}/level` | 현재 레벨 및 이력 조회 |

### SSE 이벤트 흐름

```
[ai_start] → [level_assessment] → [level_adjustment?] → [ai_chunk × N] → [evaluation] → [ai_complete]
```

| SSE 이벤트 | 내용 |
|-----------|------|
| `level_assessment` | 어휘·문법·유창성 기반 레벨 판단 결과 |
| `level_adjustment` | 레벨 변경 시 발생 (from → to) |
| `ai_chunk` | AI 응답 텍스트 토큰 단위 스트리밍 |
| `evaluation` | 사용자 문장 점수(0~100) + 문법/뉘앙스 피드백 |

## 4. Implementation Progress (현재까지의 성과)

### 구현 완료 기능

- **인증 시스템**: Google OAuth → AWS Cognito → JWT 검증 전체 흐름 구현. dev 프로필에서 인증 우회 설정으로 빠른 개발 가능
- **채팅방 관리**: 생성/수정/삭제/고정 CRUD. 유저당 캐릭터 1개 제한 + 소유권 검증
- **메시지 시스템**: 텍스트·이미지 메시지 저장 및 히스토리 페이지네이션 조회
- **AI SSE 스트리밍**: GPT-4o API 스트리밍 → WebFlux Flux → SSE로 실시간 전달
- **레벨 판단 시스템**: 메시지마다 어휘·문법·유창성 자동 분석, 신뢰도(confidence) 산출
- **자동 레벨 조절**: Redis에 최근 5개 점수 캐싱 → 연속 3회 ≥90점 레벨 업, 연속 2회 ≤50점 레벨 다운
- **문장 평가 시스템**: 문법 오류, 뉘앙스, 긍정 포인트 포함한 구조화된 피드백 (0~100점)
- **공통 응답 규격**: `ApiResponse<T>` 래퍼 + 전역 예외 핸들러로 일관된 에러 응답
- **DB 스키마 관리**: Flyway 마이그레이션으로 버전 관리

### 핵심 데이터 모델

```
users (1)
  └── chat_rooms (N)         ← user_id + character_id UNIQUE
        ├── chat_messages (N) ← senderType, evaluation, score
        └── level_assessments (N) ← confidence, indicators
```

## 5. Deployment & Future Roadmap (향후 계획)

### AWS 클라우드 배포 전략

| 서비스 | 용도 | 설정 |
|--------|------|------|
| **EC2** (t3.medium) | Spring Boot 백엔드 실행 | Amazon Linux 2023, JDK 25, IAM Role |
| **S3** (fluento-images) | 채팅 이미지 저장 | 퍼블릭 차단, Pre-signed URL, SSE-S3 암호화 |
| **Cognito** (fluento-users) | Google 소셜 로그인 + JWT 발급 | Google OAuth 연동, Access Token 1h |
| **PostgreSQL** | 영구 데이터 저장 | EC2 내 또는 RDS 전환 고려 |
| **Redis** | 레벨 조절 점수 캐시 | EC2 내 또는 ElastiCache 전환 고려 |

**배포 순서**: Cognito 설정 → S3 버킷 생성 → IAM Role 생성 → EC2 인스턴스 생성 및 JAR 배포

### 단계별 개발 로드맵

| 단계 | 목표 | 주요 작업 |
|------|------|----------|
| **1단계** (현재) | 백엔드 핵심 기능 완성 | SSE 스트리밍, 레벨 시스템, 평가 API 구현 ✅ |
| **2단계** | AWS 배포 | EC2/S3/Cognito 설정, prod 환경 배포, HTTPS 적용 |
| **3단계** | 프론트엔드 연동 | 모바일/웹 클라이언트 API 연동, SSE 클라이언트 구현 |
| **4단계** | 고도화 | ALB + HTTPS, RDS/ElastiCache 전환, 오답노트 기능 |
| **최종** | 서비스 완성 | 성능 최적화, 이미지 메시지 기능, 캐릭터 확장 |
