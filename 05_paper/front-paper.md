# Fluento: 사용자 경험을 고려한 AI 영어 학습 플랫폼 프론트엔드 개발

## I. 서론

영어 학습의 효율성을 높이기 위해서는 반복적인 실습과 즉각적인 피드백이 필수적이다. 그러나 대면 학습은 시간과 장소의 제약이 있고, 기존 온라인 학습 플랫폼은 단순한 문제 풀이에 그치는 경우가 많다. 이러한 문제점을 해결하기 위해 본 연구에서는 AI 캐릭터와의 실시간 대화를 통한 영어 학습 플랫폼 Fluento의 프론트엔드를 개발하였다.

Fluento는 사용자가 직접 생성한 AI 캐릭터와 채팅하며 자연스러운 영어 회화를 연습하고, 실시간으로 문법 교정을 받을 수 있는 특징을 제공한다. 이러한 상호작용적 학습 방식은 사용자의 몰입도를 높이고, 실질적인 의사소통 능력을 향상시킨다.

본 연구의 목표는 다음과 같다:
- 직관적인 사용자 인터페이스(UI)를 통한 편리한 접근성 제공
- 사용자 맞춤형 AI 친구 캐릭터 생성 기능
- 실시간 문법 교정 및 피드백 시스템 구현
- 보안과 신뢰성을 갖춘 사용자 인증 체계

## III. 주요 페이지 기능 및 설계

### 3.1 로그인 페이지

**목적 및 설계**

로그인 페이지는 기존 사용자의 접근을 관리하고, 개인 학습 데이터를 보호하는 첫 번째 보안 관문이다. 사용자는 이메일 주소와 비밀번호를 입력하여 시스템에 접근한다.

화면 상단에는 Fluento 로고가 배치되어 브랜드 아이덴티티를 강조한다. 중앙의 하얀색 카드 형태 로그인 영역에는 두 개의 입력 필드가 있다. 첫 번째는 이메일 입력창으로, 사용자의 고유 식별자 역할을 한다. 두 번째는 비밀번호 입력창으로, 입력한 문자를 보호하기 위해 기본적으로 점(●) 형태로 표시된다.

**보안 기능**

비밀번호 필드의 우측에는 "보기/숨김" 토글 버튼이 위치한다. 이 버튼을 클릭하면 비밀번호를 일시적으로 일반 텍스트로 표시하여, 사용자가 입력 오류를 확인할 수 있도록 했다. 이는 사용자 편의성과 보안의 균형을 맞추기 위한 설계이다.

로그인 버튼은 보라색(#4E3473)으로 강조되어 주요 행동을 유도한다. 로그인 완료 시 사용자의 이메일 정보가 로컬 저장소에 임시 저장되어 이후 사용자 맞춤형 화면을 제공한다.

**추가 기능**

하단에는 "또는 소셜 계정으로 로그인" 옵션이 제공되며, Google과 Apple 계정을 통한 빠른 로그인 기능도 준비되어 있다. 페이지 하단의 링크를 통해 신규 사용자는 회원가입 페이지로, 기존 사용자는 비밀번호 찾기 기능으로 이동할 수 있다.

**그림 1. 로그인 화면**

### 3.2 회원가입 페이지

**목적 및 설계**

회원가입 페이지는 신규 사용자가 안전하게 계정을 생성하는 과정을 관리한다. 사용자 정보의 정확성과 보안을 동시에 보장하도록 설계되었다.

화면은 단계적 입력 구조로 구성되어 있다. 첫 번째 단계에서는 사용자 이름을 입력받는다. 두 번째 단계는 이메일 인증 절차로, 이메일 주소 입력 후 "인증요청" 버튼을 클릭하면 인증번호를 발송한다.

**이메일 인증 시스템**

인증번호 발송 후 새로운 입력 필드가 나타난다. 이 필드에는 사용자가 받은 6자리 인증번호를 입력한다. 인증번호 하단에는 유효 시간(03:00)을 표시하여 사용자에게 시간 제약을 명확히 알린다. 이러한 타이머는 보안 위협(예: 무차별 대입 공격)으로부터 계정을 보호한다.

세 번째와 네 번째 단계는 비밀번호 설정이다. 첫 번째 입력창에는 새 비밀번호를, 두 번째 입력창에는 비밀번호 확인을 입력한다. 두 입력값이 일치하지 않으면 경고 메시지가 표시되어 사용자 실수를 방지한다.

**회원가입 완료**

모든 필드가 올바르게 입력되면 "가입하기" 버튼이 활성화된다. 가입 완료 후 자동으로 로그인 페이지로 이동하여, 사용자가 곧바로 서비스를 이용할 수 있도록 설계했다.

**그림 2. 회원가입 화면**

### 3.3 홈 페이지

**목적 및 설계**

홈 페이지는 사용자의 일일 학습 목표를 관리하고 학습 동기를 부여하는 대시보드 역할을 한다. 페이지 진입 시 사용자를 맞이하는 첫 번째 화면으로, 긍정적인 학습 경험을 제공하도록 설계했다.

화면 상단에는 "오늘의 목표에요"라는 제목이 있고, 그 아래 체크박스 형태의 목표 목록이 나열된다. 기본 목표는 "10분간 영어학습 진행하기!", "단어 20개 완벽하게 암기하기!", "오답노트 다시 확인하기!" 등으로 사용자의 학습 의지를 고취시킨다.

**목표 추적 기능**

각 목표 옆의 체크박스를 클릭하면 해당 항목이 완료된 것으로 표시된다. 완료된 항목은 회색으로 변하고 텍스트에 취소선이 나타나 시각적 피드백을 제공한다. 이는 사용자의 성취감을 높이고 학습 지속력을 강화한다.

**학습 콘텐츠**

페이지 중단부에는 수평선으로 구분되어 두 개의 섹션이 있다. 좌측에는 "오늘의 표현 하나 배워보세요"라는 제목 아래 일일 영어 표현과 해석이 제시된다. 예를 들어 "Don't even think about it." (꿈도 꾸지마!)와 같은 실용적인 표현을 제공한다.

우측에는 "동기부여 가득!"이라는 제목 아래 동기 부여 콘텐츠가 표시되도록 설계했다. 향후 이 영역에 AI가 생성한 격려 메시지나 학습 통계를 추가할 계획이다.

**그림 3. 홈페이지 화면**

### 3.4 친구 목록 페이지

**목적 및 설계**

친구 목록 페이지는 사용자가 생성한 AI 캐릭터를 관리하고 선택하는 중추 기능이다. 다양한 필터링과 검색 기능을 통해 사용자가 원하는 친구를 빠르게 찾을 수 있도록 설계했다.

페이지 상단에는 "모든 친구" 와 "즐겨찾는 친구" 두 개의 탭 버튼이 있다. 사용자는 이 탭을 클릭하여 보고 싶은 친구 목록을 필터링할 수 있다. 탭 선택 상태는 버튼의 색상 변화(보라색 강조)로 시각적으로 표시된다.

상단 우측에는 검색 입력 창이 위치한다. 사용자는 친구의 이름 일부만 입력하면 실시간으로 해당하는 친구만 화면에 표시된다.

**친구 항목 구성**

각 친구는 카드 형태로 표시된다. 좌측에는 친구의 프로필 이미지(원형)가 있고, 이미지 좌상단에는 즐겨찾기 친구인 경우 별(⭐) 아이콘이 표시된다. 

프로필 우측에는 친구의 정보가 나열된다. 상단에는 친구 이름(예: "Alex", "Mina")이 큰 글씨로 표시되고, 하단에는 설명(예: "Professor / male / kind, strict")이 작은 글씨로 표시된다.

각 친구 항목의 우측 끝에는 세로 점(⋮) 아이콘이 있다. 이를 클릭하면 드롭다운 메뉴가 나타나 "즐겨찾기 등록/해제", "친구 삭제" 옵션을 제공한다.

**모달 창 (상세 정보)**

친구 항목을 클릭하면 전체 화면을 덮는 모달 창이 나타난다. 모달의 좌상단에는 뒤로가기 화살표가 있어 쉽게 목록으로 돌아올 수 있다.

모달 상단에는 친구의 프로필 이미지(200x200px 크기)가 크게 표시되고, 그 아래 친구의 이름과 설명이 표시된다. 

모달 하단에는 "메모" 텍스트 입력 영역이 있다. 경계선으로 구분된 이 영역에 사용자는 해당 친구에 대한 기억할 사항이나 학습 관련 메모를 작성할 수 있다. 예를 들어 "Speaking practice focus" 같은 개인 노트를 남길 수 있다.

**플로팅 버튼**

페이지 우측 하단에는 원형의 보라색 플로팅 버튼이 고정되어 있다. "+" 기호가 표시된 이 버튼을 클릭하면 새로운 AI 친구를 생성하는 커스텀 친구 페이지로 이동한다.

**그림 4. 친구 목록 페이지와 상세 모달**

### 3.5 커스텀 친구 추가 페이지

**목적 및 설계**

커스텀 친구 페이지는 사용자가 자신이 원하는 AI 캐릭터를 직접 생성하는 창작 공간이다. 사용자의 개성과 학습 선호도를 반영하여 맞춤형 학습 파트너를 만들 수 있다.

페이지 상단에는 뒤로가기 화살표와 "커스텀 하기"라는 제목이 있다. 메인 콘텐츠는 좌우 두 섹션으로 나뉜다.

**좌측: 프로필 이미지 섹션**

좌측에는 "프로필 이미지"라는 제목 아래 큰 원형 영역이 있다. 이 영역은 검은색 경계선으로 둥글게 구분되어 있으며, 클릭하면 사용자 기기의 파일 탐색기가 열려 이미지를 선택할 수 있다.

이미지를 선택하기 전에는 원형 영역 내 중앙에 큰 "+" 기호와 "이미지 업로드"라는 텍스트가 표시된다. 이미지를 선택하면 선택된 이미지가 원형으로 자동 조정되어 표시된다. 이를 통해 사용자는 선택한 이미지가 어떻게 표시될지 실시간으로 미리볼 수 있다.

영역 아래에는 "원을 클릭하여 사진을 선택하세요."라는 안내 텍스트가 있어 사용자 편의성을 높인다.

**우측: 정보 입력 섹션**

우측 섹션에는 친구의 정보를 입력하는 필드들이 배열되어 있다.

첫 번째 행에는 "이름"과 "성별" 필드가 나란히 있다. 이름 필드는 텍스트 입력이고, 성별 필드는 "남성/여성" 두 개의 버튼으로 구성되어 있다. 선택된 성별은 보라색으로 강조되어 현재 선택 상태를 명확히 보여준다.

다음에는 "직업" 필드(예: 교수, 친구, 이웃)와 "성격" 필드(예: 친절함, 엄격함)가 있다. 이 두 필드는 사용자가 원하는 캐릭터의 배경과 성향을 정의한다.

그 아래에는 "메모" 텍스트 영역이 있다. 여러 줄의 입력이 가능한 이 영역에 사용자는 "이 캐릭터에 대해 기억해야 할 내용을 적어주세요."라는 안내에 따라 캐릭터에 대한 상세 정보나 학습 목표를 작성할 수 있다.

**생성 버튼**

페이지 하단에는 큰 보라색 버튼 "AI 친구 생성하기"가 있다. 이 버튼을 클릭하면 입력한 모든 정보가 검증된다. 이름과 직업이 입력되지 않으면 경고 메시지가 표시되어 필수 항목을 입력하도록 안내한다. 모든 필드가 올바르게 입력되면 친구가 생성되고, 사용자는 자동으로 친구 목록 페이지로 이동하여 새로 생성된 친구를 확인할 수 있다.

**그림 5. 커스텀 친구 생성 화면**

### 3.6 채팅 페이지

**목적 및 설계**

채팅 페이지는 사용자와 AI 캐릭터 간의 영어 대화를 실현하는 핵심 기능이다. 자연스러운 대화 환경과 함께 실시간 문법 교정을 제공하여 학습 효과를 극대화한다.

페이지 상단에는 헤더가 있다. 좌측에는 뒤로가기 화살표가 있어 채팅 목록으로 돌아갈 수 있다. 중앙에는 대화 중인 AI 캐릭터의 이름이 표시되고, 우측은 향후 설정 옵션을 위한 공간으로 예약되어 있다.

**실시간 교정 창**

헤더 아래에는 보라색으로 강조된 "실시간 교정 창"이 배치되어 있다. 이 영역은 사용자가 입력한 영어 문장의 문법 오류를 즉시 표시한다.

교정 창의 구조는 다음과 같다:
- 상단: "실시간 교정 창" 제목
- 중앙: 사용자의 원본 문장(예: "Today I has a very good day.") → 수정된 문장(예: "Today I had a very good day.")
- 하단: 교정 이유 설명(예: "The past tense of 'have' is 'had'.")

원본 문장은 취소선으로 표시되어 오류임을 시각적으로 표현하고, 수정된 문장은 강조되어 올바른 표현을 명확히 한다.

**채팅 영역**

교정 창 아래는 메시지가 누적되는 채팅 영역이다. 화면을 따라 자동으로 스크롤되어 항상 최신 메시지가 화면 하단에 보이도록 설계했다.

**메시지 표시 방식**:
- 사용자 메시지: 화면 우측에 보라색 말풍선으로 표시
- AI 메시지: 화면 좌측에 흰색 말풍선으로 표시되며, 좌측에는 "AI 프로필" 아바타 원이 함께 표시

각 메시지에는 전송 시간(예: "10:00 AM")이 함께 표시된다.

**입력 영역**

페이지 하단에는 메시지 입력 영역이 있다. 좌측에는 파일이나 이미지를 첨부하기 위한 "+" 버튼이 있고(향후 기능), 그 옆에는 보라색 입력 바가 있다. 

입력 바 내부에는 "채팅 입력" 플레이스홀더 텍스트가 있고, 우측에는 흰색 배경의 전송 버튼(화살표 아이콘)이 있다. 사용자가 텍스트를 입력하고 전송 버튼을 누르면 메시지가 채팅 영역에 추가되고, 1초 후 AI의 응답이 자동으로 나타난다.

**학습 효과**

이러한 설계는 사용자가 자연스럽게 영어를 입력하고, 즉각적인 피드백을 받으며, 올바른 표현을 학습하는 과정을 촉진한다. 특히 실시간 교정 창은 문법 오류를 수정하기 전에 사용자가 왜 오류가 발생했는지 이해하도록 돕는다.

**그림 6. 채팅 화면과 실시간 교정 창**

## IV. 보안 및 신뢰성

### 4.1 사용자 인증 체계

본 플랫폼은 사용자 정보 보호와 신뢰 구축을 최우선으로 한다.

**다단계 인증 절차**:
1. 회원가입: 이메일 인증번호 검증 (6자리, 3분 유효)
2. 로그인: 이메일 + 비밀번호 인증
3. 개인 학습 데이터: localStorage를 통한 세션 관리

**비밀번호 보안**:
- 입력 필드에서 비밀번호 마스킹 (●●●● 표시)
- 사용자 선택에 따른 보이기/숨기기 기능
- 회원가입 시 비밀번호 재확인을 통한 입력 오류 방지

### 4.2 향후 보안 강화 방안

**학교 인증 시스템 연동** (참고논문 사례):
- 학교의 공식 인증 시스템(KUTIS 등)과 연동하여 사용자 신원 자동 확인
- 학생증 제출 요구 기능으로 위장 계정 차단

**보안 질문 도입**:
- 채팅 파트너 생성 시 "가장 선호하는 학습 주제는?" 같은 보안 질문 설정
- 향후 계정 복구 시 보안 질문을 통한 본인 인증
- 실제 소유자만이 답할 수 있는 질문으로 보안 강화

**개인정보 보호**:
- 민감한 개인정보(연락처, 학번 등)는 필요할 때만 표시
- 채팅 기록 암호화 저장으로 개인 학습 내용 보호

## V. UI/UX 설계 특징

### 5.1 색상 및 시각 설계

- **주색상**: #4E3473 (보라색) - 전문성과 신뢰성을 나타내는 브랜드 색상
- **배경색**: #ffffff (흰색) - 깔끔함과 높은 가독성
- **강조색**: 파란색 및 빨간색 - 경고 및 중요 정보 표시
- **그림자 효과**: 부드러운 보라색 그림자로 깊이감 표현
- **모서리**: 둥근 모서리(8-30px)로 친화적 분위기 조성

### 5.2 사용성 설계

- **직관적 네비게이션**: 사이드바로 모든 주요 페이지에 빠르게 접근
- **조건부 헤더 표시**: 친구 목록과 채팅 페이지에서는 헤더를 숨겨 집중력 강화
- **플로팅 버튼**: 주요 행동(친구 추가)을 항상 접근 가능하게 배치
- **모달 창**: 상세 정보를 현재 페이지 맥락 내에서 확인 가능
- **실시간 피드백**: 검색, 필터링, 입력 값 변화에 즉각 반응

### 5.3 반응형 디자인

- **레이아웃**: Flexbox 기반으로 다양한 화면 크기에 적응
- **접근성**: 키보드 네비게이션 지원 및 스크린 리더 호환성 고려

## VI. 결론 및 향후 연구

### 6.1 개발 성과

본 연구를 통해 다음과 같은 성과를 도출했다:

1. **사용자 중심 인터페이스**: 초보 학습자도 쉽게 사용할 수 있는 직관적 설계
2. **다단계 보안 체계**: 이메일 인증, 비밀번호 관리, 향후 보안 질문 도입으로 신뢰 구축
3. **상호작용적 학습**: 실시간 채팅과 즉각적 문법 교정으로 학습 몰입도 증대
4. **맞춤형 경험**: 사용자 정의 AI 캐릭터로 개인화된 학습 환경 제공

### 6.2 현재 제약사항

- 백엔드 API 연동 미완료 (데이터베이스 저장 기능 제한)
- 실제 AI 문법 교정 엔진 미구현 (시뮬레이션 단계)
- 실시간 채팅 웹소켓 연결 미구현
- 학교 인증 시스템 연동 미비

### 6.3 향후 개발 계획

**1단계: 백엔드 통합**
- Node.js/Python 기반 백엔드 API 개발
- 사용자 정보, 친구 정보, 채팅 기록 데이터베이스 구축
- 데이터베이스 연동으로 사용자 학습 이력 저장

**2단계: AI 엔진 적용**
- 실제 자연어 처리(NLP) 모델 도입으로 자동 문법 교정
- ChatGPT 또는 Google Translate API를 활용한 실시간 피드백
- AI 캐릭터의 동적 응답 생성

**3단계: 실시간 통신**
- WebSocket을 활용한 양방향 통신 구현
- 다중 사용자 동시 채팅 지원 (향후)
- 푸시 알림 시스템 추가

**4단계: 보안 강화**
- 학교 인증 시스템(KUTIS 등) 연동
- 보안 질문을 통한 추가 본인 인증 절차
- 민감한 개인정보 암호화 및 블러 처리
- 채팅 기록 암호화 저장

**5단계: 확장 기능**
- 사용자 간 상호작용 및 커뮤니티 기능
- 학습 통계 및 성과 추적 대시보드
- 게임화 요소 (뱃지, 포인트, 랭킹)
- 모바일 앱(React Native)으로 확장
